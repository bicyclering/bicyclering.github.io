webserver xdebug

PhpStrom

install xdebug

matching debug-version

  1. Installation Wizard

  2. copy phpinfo() information

  3. Download xdebug-x.x.x.tgz

  4. Install the pre-requisites for compiling PHP extensions.On your Mac, we only support installations with ‘homebrew’, and brew install php && brew install autoconf should pull in the right packages.

  5. Unpack the downloaded file with tar -xvzf xdebug-x.x.x.tgz

  6. Run: cd xdebug-x.x.x

  7. Run: phpize (See the FAQ if you don’t have phpize).

    As part of its output it should show:

    Configuring for:
...
Zend Module Api No:      20200930
Zend Extension Api No:   420200930

  1. If it does not, you are using the wrong phpize. Please follow this FAQ entry and skip the next step.

    check xdebug-x.x.x path

    for example: cd /xxxxx/xxxxx/Documents/xdebug-x.x.x/xdebug-x.x.x

  2. Run: ./configure

  3. Run: make

  4. Run cp modules/xdebug.so /Applications/XAMPP/xamppfiles/lib/php/extensions/no-debug-non-zts-20200930

  5. Update /Applications/XAMPP/xamppfiles/etc/php.ini and add the line:zend_extension = xdebug

  6. Restart the Apache Webserver

conig webserver xdebug

配置php.ini 

    [xdebug]
zend_extension = xdebug
xdebug.idekey  = PHPSTROM
